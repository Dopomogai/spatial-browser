# Canvas Menu Closes On WebViews
**Issue:** User reported: "almost there - everywhere exept for if you click inside the tabs" (Regarding context menu dismissal).
**Resolution:** The global blur event or document-level click listener needs to be expanded. When a user clicks *inside* an `iframe` or Chromium `webview`, `onPointerDown` on the canvas `div` won't trigger because the sandbox completely steals it. Add an `App.tsx` global native listener for blurring the host, or update the `BrowserWidgetNode` to dispatch a global `close-context-menu` event `onPointerDown`. Alternatively use `window.addEventListener('blur', setContextMenuOpen(false))` if it triggers reliably upon `webview` native focus.
