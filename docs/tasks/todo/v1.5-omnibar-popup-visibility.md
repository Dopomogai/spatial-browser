# Task: Omnibar Search Popup Not Appearing

## Problem Statement
The user reported that the search popup menu (Omnibar) does not appear when invoked. Other spawn actions like "Add Text Note" from the canvas work correctly, but summoning the URL entry field fails to display anything on the screen.

## Hypothesis
1. The `<Omnibar />` component is either unmounted or conditionally blocked by z-index logic or visibility hooks at the top app layer.
2. The trigger event (`open-omni-search` from the Top Bar, or `setOmnibarOpen` from the Context Menu) is failing to pass coordinates or properly mutate the Zustand store (`isOmnibarOpen: true`).

## Action Plan
1. Check `src/renderer/src/App.tsx` and `src/renderer/src/components/canvas/SpatialCanvas.tsx` to verify exactly where `<Omnibar />` is structurally mounted.
2. If it's missing or wrapped inside a blocked conditional, restore it to a global floating layer with an extremely high `z-index`.
3. Check `src/renderer/src/store/useCanvasStore.ts` to ensure `setOmnibarOpen(isOpen, position)` functions.
4. Verify the `Search` button in `TopTabBar.tsx` correctly calls `window.dispatchEvent(new CustomEvent('open-omni-search'))` and that an event listener actually exists in `SpatialCanvas.tsx` to catch it.
5. Commit to main.
