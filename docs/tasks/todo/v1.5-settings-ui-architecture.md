# Task: Settings UI Rendering Crash

## Problem Statement
Clicking the "Settings" button in the TopTabBar, or anywhere that spawns the Settings node, causes the application to throw Node/Vite isolation errors or renders a blank `about:blank#settings` page. The React-based settings component fails to mount properly.

## Hypothesis
1. The canvas node is treating the Settings panel exactly like a standard Webview (`BrowserWidgetNode.tsx`), aggressively attempting to load the `url` as a Chromium target.
2. We need a conditional fork in the `BrowserWidgetNode.tsx` rendering block: if the URL passed is strictly `about:blank#settings`, it should physically bypass the `<webview>` Chromium element and render the localized React `<SettingsWidget />` component.

## Action Plan
1. Inspect `src/renderer/src/components/shapes/BrowserWidget/BrowserWidgetNode.tsx`.
2. Inspect `src/renderer/src/components/settings/SettingsWidget.tsx` (or similar file) to verify the component exists.
3. Import the Settings component into `BrowserWidgetNode`.
4. Create a conditional return: `if (data.url === 'about:blank#settings') return <SettingsWidget />;` directly inside the node so it acts as standard HTML rather than a sandboxed Chromium instance.
5. Commit fix to main.
