# Top Bar Tab Hide / Re-design
**Issue:** The UI requests require the large Tab browser widgets to be optionally hidden, leaving a condensed top bar containing the search interface, settings gear, and history buttons.
**Hypothesis:** Conditional rendering applied at the parent level (`TopTabBar.tsx` and `App.tsx`) via an `isTopTabBarVisible` Zustand flag obscures the canvas tabs layout without tearing down the nodes array in memory.
**Actionable Fix:** 
1. Map `isTopTabBarVisible` to the Zustand `useCanvasStore.ts` layer, triggering via Settings UI interaction.
2. Redesign the HTML for individual `TabItems` mapped in the `TopTabBar` sequence. The request is specifically for "nice and beautiful bubbles," meaning full `filter: drop-shadow`, `border-radius: 9999px`, and `backdrop-blur`.
3. Build logic to support tab grouping (a colored outline box enclosing multiple Tab Items), extending visual organization mechanics.
