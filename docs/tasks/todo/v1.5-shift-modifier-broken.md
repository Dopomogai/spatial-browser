# Shift Key Modifier Broken
**Issue:** Holding "Shift" works initially, but breaks (stops panning the canvas) as soon as the user clicks inside a tab/webview.
**Resolution:** When the user clicks inside a `<webview>`, the Chromium instance steals focus and traps the keyboard events natively. The Main Electron process must register a global `globalShortcut` or use `webContents.on('before-input-event')` to intercept the Shift key globally and pipe it back to the renderer, OR the webview preload script must actively forward `keydown` events back up to the host window container.
