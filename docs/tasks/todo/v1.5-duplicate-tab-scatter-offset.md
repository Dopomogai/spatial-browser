# Task: Duplicate Tab Scatter Offset

## Problem Statement
When a user clicks "Duplicate" on an existing Tab window, the Zustand store correctly replicates the AppNode data. However, the new tab is placed at the exact same physical [x,y] coordinates as the original tab. This causes the UI to appear completely unchanged until the user manually clicks and drags the top window away to reveal the original.

## Hypothesis
The duplicate function in `useCanvasStore.ts` does not mathematically apply an offset (e.g., +20px on the X and Y axes) when generating the duplicate payload. 

## Action Plan
1. Inspect the duplicate logic (likely within `useCanvasStore.ts` or triggered from the Node Context Menu).
2. Locate the cloned node's positioning logic.
3. Inject a standard offset (+20px relative to the source node's `x` and `y`).
4. Ensure the duplicate is still rendered at a high `z-index`.
5. Commit to main.
