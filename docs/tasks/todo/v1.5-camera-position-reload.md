# Canvas Position Not Actually Restored On Reload
**Issue:** The user stated "I was wrong - this doesnt work at all properly - just same position almost it seems" regarding `lastViewport` reloading.
**Resolution:** Look at `useCanvasStore.loadInitialState()`. It calls `set({ ...stored, isOmnibarOpen: false, omnibarPosition: null } as any)`. However, `ReactFlow` requires the `defaultViewport` *prop* directly on `<ReactFlow />` OR an explicit `useEffect` containing `setViewport(lastViewport)` after the graph renders if `defaultViewport` isn't mounting asynchronously. Ensure `<ReactFlow defaultViewport={lastViewport} />` receives the *loaded* data synchronously, or explicitly trigger `setViewport` upon component mount from Zustand.
