# Cmd+Z / Cmd+X Keyboard Binding Clarification
**Issue:** User clarified that Undo (`Cmd+Z`) and Redo (`Cmd+Shift+Z`) triggers need immediate mapping, alongside Cut (`Cmd+X`). The previous fix did not effectively cover all use cases because Chromium steals focus from `App.tsx` global listeners.
**Hypothesis:** `keydown` events mapped locally in `App.tsx` are masked by `<webview>` focus states. 
**Actionable Fix:** Shift the `e.metaKey && e.key === 'z'` listeners down into the Electron `ipcMain` layer utilizing `webContents.on('before-input-event')`, capturing the keystroke natively ahead of the Chromium render pipeline. Distinguish `e.shift` accurately for Redo vs Undo. Route `Cmd+X` natively via `webContents.cut()`.
