# Canvas Position Not Restored On Reload
**Issue:** When the application is reloaded, the user's previous exact canvas location (the camera viewport position) is not restored, likely resetting to a default coordinate (0,0) or bounding box fit.
**Hypothesis:** While the `nodes` array is persisted effectively in the Zustand store (saving widget positions relative to the canvas), the `viewport` state (`x`, `y`, `zoom`) is either not being written to the persisted local storage, or `ReactFlow` is not initializing with the saved `defaultViewport` on startup. 
**Actionable Fix:** 
1. Map `viewport` to the `useCanvasStore.ts` interface and update it periodically or on `onMoveEnd`.
2. Inject the saved viewport directly into `<ReactFlow defaultViewport={...} />` inside `SpatialCanvas.tsx` during the initial mount cycle.
