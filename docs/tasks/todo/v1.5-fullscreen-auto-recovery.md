# Fullscreen Native Trapping & Minimap Overlap

## What we need
The "Fullscreen" green button on tabs requires two clicks to size perfectly. When it *is* full screen, the `<MiniMap>` and `<TopTabBar>` are still visible and block the website UI. There is also no way to auto-recover/minimize back cleanly from fullscreen.

## Why
True fullscreen means the website takes over 100% of the glass UI, and the user must be able to escape it intuitively without getting trapped. The double-click behavior is jarring UI friction.

## How to do it
1. Fix the double-click React Flow rendering race condition in `SpatialCanvas.tsx` by ensuring the camera `zoom: 1` `setCenter` coordinates wait for the `updateWidgetData` width/height DOM mutations to resolve (or simply fire `setCenter` simultaneously).
2. Inject a boolean `isFullScreenMode` into `useCanvasStore.ts`. When true, unmount `<TopTabBar>` and `<MiniMap>` completely. 
3. Ensure the Red/Yellow/Green macOS buttons stay visible *above* the webview so the user can click Green again to shrink back seamlessly.
