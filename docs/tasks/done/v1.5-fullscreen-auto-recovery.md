# Task: Fullscreen Auto-Recovery & React Flow Maximize

## Problem Statement
Clicking the "Fullscreen" button inside a `BrowserWidgetNode.tsx` throws an `Uncaught TypeError: Cannot read properties of undefined (reading 'send')` at `onClick`. Furthermore, it completely fails to cover the React Flow canvas or maximize the underlying standard Electron browser window seamlessly.

## Hypothesis
1. The `<Maximize2>` button relies on the legacy `window.ipcRenderer.send('toggle-fullscreen')` directly instead of leveraging `window.electron.ipcRenderer.send('toggle-fullscreen')` which context isolation requires in Vite+Electron.
2. Even if the IPC is fixed, React Flow must logically adjust its `position` and `style` to span `100vw` and `100vh` at `{x: 0, y: 0}`, and dynamically set the Node to the absolute top `z-index`.

## Action Plan
1. Fix the `ipcRenderer` crash inside `BrowserWidgetNode.tsx`'s fullscreen header button by ensuring it uses `window.electron.ipcRenderer.send`.
2. Inspect `useCanvasStore.ts`. Expose a `toggleFullscreen(id: string)` method that mutates a specific node to `width: window.innerWidth`, `height: window.innerHeight`, `x: 0`, `y: 0`, and caches its previous positional coordinates inside the node payload.
3. If clicked again (exit Fullscreen), the store should retrieve these cached coords and pop it back to its original state.
4. Commit to main.
