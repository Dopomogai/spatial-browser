# Task: Visually Distinguish Disabled Undo/Redo Buttons

## Problem Statement
The Undo and Redo features mechanically function, but the user cannot visually distinguish when the buttons are enabled versus disabled. The icons look identically styled regardless of the history stack state.

## Hypothesis
The visual state (opacity, cursor type, hover effects) is not dynamically bound to the `undoStack.length` and `redoStack.length` inside the Top Bar component.

## Action Plan
1. Inspect the `TopTabBar.tsx` component.
2. Subscribe to the `undoStack` and `redoStack` arrays from the Zustand store (`useCanvasStore.ts`).
3. Apply conditional Tailwind CSS classes (e.g., `opacity-30 cursor-not-allowed` and strip `hover:bg-*` effects) when the respective stack length is `0`.
4. Commit the fix directly to main.
