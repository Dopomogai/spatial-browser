# Shift Key Modifier - Webview Focus Steal
**Issue:** The `before-input-event` fix works initially, but the user reports that holding shift breaks again once they actually click inside an active webpage tab.
**Hypothesis:** Chromium webcontents `before-input-event` might only be firing when the overall webview frame has focus, but the iframe/document inside the webview traps it deeper, or the IPC message `shift-state-change` is failing to clear correctly internally on blur/mouse click. 
**Actionable Fix:** Re-evaluate `src/main/index.ts`. Add a global `app.on('browser-window-blur')` or track webview `focus` and `blur` events explicitly in `<BrowserWidgetNode>` to manually override `isShiftHeld` globally.
