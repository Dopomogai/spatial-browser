# Drag Release Lag / Performance Input Latching
**Status:** In Review

**Issue:** Extreme input lag, especially when dragging with 3+ webviews open on the canvas. The drop action is laggy, and tabs sometimes stay "latched" to the cursor.

**Resolution:** This is a severe rendering/state cycle performance bottleneck. React Flow is re-rendering the entire canvas matrix aggressively on every pixel movement of an active webview, causing Chromium IPC congestion. 

*   **Actionable Fix 1:** Wrap the `BrowserWidgetNode` internals in `React.memo` or decouple the `w`/`h` dimensions out of the global Zustand React Flow nodes array during active drags mapping to local state instead until `onResizeEnd`. (DONE)
*   **Actionable Fix 2:** Optimize the `ReactFlow` `<Background/>` to avoid redraws on pointer moves.
